# 家庭工作台：当前架构基线

## 单家庭边界

本系统当前只服务一个家庭。`SiteSetting` 保存工作台名称、默认本位币和时区；`Family` 暂时作为兼容层保留，以避免一次迁移同时改写所有生产外键。管理后台禁止新建第二个家庭和删除现有家庭。

物理删除 `Family` 表及各模块的 `family_id` 不属于本阶段。应在生产数据通过一致性检查、所有查询改用站点配置后，另行设计迁移和回滚方案。

## 成员、账户归属与操作者

- `FamilyMember` 表示资产或账目的归属人，也负责登录身份和角色。
- 家庭内的有效成员默认可以查看所有成员的数据，并可以替其他成员录入数据。
- 业务记录中的 `member`/`account` 表示数据归属，不表示实际操作人。
- `created_by`/`updated_by` 记录实际登录操作者，从而支持“A 登录后录入 B 的账户交易”。
- `viewer` 只读；`member` 和 `admin` 可修改业务数据；未绑定或已停用的登录用户不能进入业务页面。

## 数据来源与后续阶段

交易流水是可审计事实，当前持仓是由交易变化维护的派生状态，快照是特定时点的估值结果。

## 账户与投资核算口径

- `BankAccount` 是账户名称、归属成员、地区、脱敏账号和有效状态的唯一来源。
- `supports_investment` 和 `supports_ipo` 明确账户用途；业务逻辑不再读取中文类型名称或备注判断用途。
- `InvestmentAccount` 只作为投资扩展，与 `BankAccount` 一对一关联。
- `InvestmentTransaction` 是买入、卖出、股息、利息、费用、税费和 IPO 交易的唯一事实源。
- IPO 表中的卖出数量、日期、价格、费用和利润是只读汇总投影，由显式关联交易回填，不作为交易输入。
- `InvestmentPosition` 每个账户和标的只有一行，表示当前持仓；重建时从全部有效交易流水计算。
- `PortfolioSnapshot` 保存历史总额，`PortfolioSnapshotPositionLine` 冻结现金及股票、ETF、债券等各资产类型的价格、汇率和估值。
- 首页和快照使用同一估值服务。快照命令只使用运行时已有的最新行情缓存，不在页面 GET 请求中抓行情或写数据库。

汇率口径：交易盈亏先按交易原币计算；组合查看和快照使用估值日可取得的最近汇率折算，禁止缺失汇率时按 1:1 合并。当前不单独拆分“证券收益”和“汇兑收益”，现金换汇应通过独立现金流水记录；如未来需要单独展示汇兑损益，再增加交易日冻结汇率和成本币种明细。

外部行情/汇率定时抓取和 AI Agent 的只读工具边界仍属于后续阶段。

## 上线前只读检查

在数据库备份后、执行迁移前后分别运行：

```bash
python manage.py audit_household_data
```

该命令只读取数据，检查家庭数量、成员/账户归属错配和重复当前持仓，不自动修复生产数据。
