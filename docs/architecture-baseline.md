# 家庭工作台：当前架构基线

## 单家庭边界

本系统当前只服务一个家庭。`SiteSetting` 保存工作台名称、默认本位币和时区；`Family` 暂时作为兼容层保留，以避免一次迁移同时改写所有生产外键。管理后台禁止新建第二个家庭和删除现有家庭。

物理删除 `Family` 表及各模块的 `family_id` 不属于本阶段。应在生产数据通过一致性检查、所有查询改用站点配置后，另行设计迁移和回滚方案。

## 成员、账户归属与操作者

- `FamilyMember` 表示资产或账目的归属人，也负责登录身份和角色。
- 家庭内的有效成员默认可以查看所有成员的数据，并可以替其他成员录入数据。
- 业务记录中的 `member`/`account` 表示数据归属，不表示实际操作人。
- `created_by`/`updated_by` 记录实际登录操作者，从而支持“A 登录后录入 B 的账户交易”。
- `viewer` 只读；`member` 和 `admin` 可修改业务数据；未绑定或已停用的登录用户不能进入业务页面。

## 数据来源与后续阶段

交易流水是可审计事实，当前持仓是由交易变化维护的派生状态，快照是特定时点的估值结果。行情刷新、快照价格冻结、IPO 与投资组合统一记账，以及 AI Agent 的只读工具边界仍为后续阶段，本轮不实现。

## 上线前只读检查

在数据库备份后、执行迁移前后分别运行：

```bash
python manage.py audit_household_data
```

该命令只读取数据，检查家庭数量、成员/账户归属错配和重复当前持仓，不自动修复生产数据。
