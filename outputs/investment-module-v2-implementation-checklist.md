# 投资板块 v2 实施计划清单

来源：`投资板块设计草稿v2.docx`

目标：逐步建设一个以交易流水为事实来源，覆盖持仓核算、交易日志、标的研究、AI 交易计划、资产配置和策略回测的家庭投资工作台。

## 一、实施原则

- [ ] 交易流水是持仓、成本和盈亏的唯一事实来源，不再依赖人工重复填写持仓结果。
- [ ] `InvestmentPosition` 作为可重建的最新持仓汇总，历史趋势进入独立快照表。
- [ ] 所有金额保留原币，同时按交易日或快照日汇率折算家庭本位币。
- [ ] 外部行情、财务、新闻和 AI 结论都记录来源、采集时间和数据版本。
- [ ] Futu 首期只做只读同步，不提供自动下单能力。
- [ ] AI 只提供分析和建议，不自动执行交易；输出必须显示依据、假设、风险和数据时间。
- [ ] 每个阶段单独迁移、测试和验收，通过后再进入下一阶段。

## 二、当前仓库基线

### 已有能力

- [x] 投资账户基础模型及新增、编辑、列表页面。
- [x] 证券标的基础模型及新增、编辑、列表页面。
- [x] 交易流水基础模型及基础 CRUD 页面。
- [x] 持仓基础模型及基础 CRUD 页面。
- [x] 投资组合快照和证券新闻缓存模型骨架。
- [x] AI 服务商、分析请求和分析结果模型骨架。
- [x] 家庭资产快照、币种和汇率相关基础能力。
- [x] 投资概览首版：最新持仓口径、资产/成本/盈亏指标及成员、账户、市场、币种、证券筛选。

### 需要改造

- [ ] 当前持仓和盈亏主要依靠人工录入，尚未由交易流水自动计算。
- [ ] 交易类型、状态、方向和资产类型仍是自由文本，需要标准化。
- [ ] 尚未处理移动平均成本、卖出成本、已实现盈亏、清仓和再次建仓。
- [ ] 多币种汇总尚未完整接入历史汇率和本位币换算。
- [ ] 数据权限目前未完整落实到家庭、成员和可见范围。
- [ ] 仪表盘仍缺少分布图、收益趋势和下钻页面。
- [ ] AI 与外部行情接口目前只有骨架。

## 三、分阶段实施清单

## 阶段 0：核算口径与数据边界

目标：先冻结账务规则，避免后续页面、图表和 AI 使用不同口径。

- [x] 明确交易类型：买入、卖出、分红、利息、手续费、税费、入金、出金、转仓、拆合股、其他。
- [x] 明确交易状态：计划中、已提交、已成交、部分成交、已取消。
- [ ] 明确投资周期：短期投机、中期波段、长期价值、事件策略。
- [x] 确认标准核算采用移动加权平均成本，同时支持切换展示摊薄成本。
- [x] 确认摊薄成本 =（剩余持仓成本 − 历史累计已实现盈亏）÷ 剩余持仓数量。
- [x] 确认买入手续费和税费计入成本，卖出手续费和税费冲减卖出收入。
- [x] 清仓后保留历史累计已实现盈亏；再次建仓时同时展示“本轮持仓”和“历史累计”指标。
- [ ] 明确现金、股票、ETF、债券和其他资产的统一分类。
- [ ] 明确本位币、交易币种、交易日汇率和当前汇率的使用规则。
- [ ] 明确 Futu 数据与人工录入冲突时的覆盖和去重规则。
- [ ] 为现有数据准备备份、校验和可重复执行的迁移方案。

验收标准：

- [ ] 形成一页可执行的核算规则说明。
- [ ] 用 5 组样例覆盖买入、加仓、部分卖出、清仓、再次建仓。
- [ ] 样例中的数量、成本、现金变动和盈亏可人工复核。

## 阶段 1：交易流水与持仓核算引擎

目标：先把“账算对”，后续所有展示和分析都建立在这一层。

- [ ] 标准化交易方向、类型、状态、来源和资产类型选项。
- [x] 补充外部流水号、现金变动、卖出成本、已实现收益率和导入来源字段。
- [x] 为外部流水号增加幂等约束，防止重复导入。
- [x] 实现按账户和证券逐笔重放的移动平均成本计算。
- [x] 实现摊薄成本计算，并与移动加权平均成本同时保存。
- [x] 为每笔卖出计算卖出成本、净收入、已实现盈亏和已实现收益率。
- [x] 自动维护最新持仓数量、平均成本、摊薄成本、市值和浮动盈亏。
- [x] 正确处理部分卖出、清仓、再次建仓及零持仓记录。
- [x] 增加从全部交易流水重建持仓的管理命令。
- [ ] 增加核算结果校验报告，标记负持仓、币种冲突和缺失价格。
- [x] 为核心核算规则建立单元测试。

验收标准：

- [ ] 手工修改或导入交易后，可一键重建持仓且结果一致。
- [ ] 同一批流水重复导入不会产生重复交易。
- [ ] 所有样例的成本和盈亏与阶段 0 规则完全一致。

## 阶段 2：持仓、交易与投资仪表盘

目标：交付日常最常用的可查询、可下钻工作台。

### 投资仪表盘

- [ ] 展示总资产、现金、持仓市值、持仓成本、浮动盈亏、已实现盈亏和累计收益率。
- [ ] 支持家庭成员、券商账户、市场、币种、资产类型和证券筛选。
- [ ] 统一按本位币汇总，同时允许查看原币金额。
- [ ] 展示账户资产分布、全账户持仓分布和资产类型分布。
- [ ] 展示按月、按年收益趋势和浮动/已实现盈亏拆分。
- [ ] 明确数据日期、行情更新时间和汇率更新时间。

### 持仓列表

- [ ] 展示代码、名称、资产类型、现价、数量、市值和移动平均成本。
- [ ] 展示浮动盈亏 1、浮动收益率、已实现盈亏和已实现收益率。
- [ ] 展示包含历史已实现盈亏的综合盈亏 2 和综合收益率。
- [ ] 展示账户内占比和全部投资账户占比。
- [ ] 清仓标的继续保留，隐藏无意义的当前持仓字段。
- [ ] 再次建仓后展示当前持仓指标及历史累计指标。

### 标的下钻

- [ ] 展示某标的在全部账户或指定账户中的交易历史。
- [ ] 每笔卖出展示卖出成本、净收入、已实现盈亏和收益率。
- [ ] 展示股价走势图并标注买入、卖出点。
- [ ] 支持从持仓行进入交易、新建计划和复盘。

验收标准：

- [ ] 从任一汇总数字可以下钻到组成它的账户、持仓或交易。
- [ ] 页面汇总值与核算引擎结果一致。
- [ ] 桌面和移动端均可使用，空数据和异常数据有明确提示。

## 阶段 3：交易日志、纪律与复盘

目标：记录“为什么交易”，并能统计行为模式。

### 交易状态与日志

- [ ] 记录核心交易逻辑及预设原因标签。
- [ ] 记录信息来源和可信度等级 A/B/C/D。
- [ ] 记录交易周期和策略类型。
- [ ] 记录恐惧、焦虑、平静、欣喜、自信、贪婪等情绪。
- [ ] 增加“计划内/计划外”“独立判断/受他人影响”等行为标签。
- [ ] 记录退出条件、目标日期、止盈位和止损线。
- [ ] 将交易信息和状态信息组合成可筛选的交易日志。

### 交易纪律

- [ ] 建立短期、中期、长期交易纪律。
- [ ] 支持启用/停用、优先级、适用市场和适用策略。
- [ ] 在交易录入和交易计划中展示相关纪律检查项。
- [ ] 记录交易是否违反纪律及违反的具体条目。

### 交易复盘

- [ ] 支持短期、中期、长期三个复盘时间点。
- [ ] 记录做对、做错、结果、教训和纪律执行情况。
- [ ] 支持将复盘教训一键转为待确认的交易纪律。
- [ ] 统计交易数、胜率、盈亏分布和月度/年度盈亏。
- [ ] 按原因、策略类型、情绪、信息可信度统计胜率和盈亏。

验收标准：

- [ ] 一笔交易可以完整追踪“计划—执行—结果—复盘—纪律沉淀”。
- [ ] 统计页面能发现不同原因、情绪和纪律执行情况的结果差异。

## 阶段 4：自选股、行情与标的研究

目标：建立 AI 分析之前的可靠数据层。

### 自选股和行情

- [ ] 建立自选股分组、关注状态、目标价和关注理由。
- [ ] 支持持仓、自选股和重要指数 ETF 的统一展示。
- [ ] 设计行情服务适配层，先支持人工更新和 Futu 只读同步。
- [ ] 同步最新价、涨跌幅、成交量、换手率和行情时间。
- [ ] 保存必要的日线历史数据，避免每次页面打开都请求外部服务。

### 新闻

- [ ] 拉取持仓和自选股相关新闻并去重。
- [ ] 保存来源、发布时间、证券关联和原文链接。
- [ ] AI 标注重要性高/中/低、利好/中性/利空及简短解读。
- [ ] 新闻结论显示模型、生成时间和可追溯原文。

### 财务与估值

- [ ] 保存近 3 年关键财务数据和数据来源。
- [ ] 保存盈利预测、估值倍数、历史估值百分位和行业估值均值。
- [ ] 支持前瞻净利润 × PE 区间的估值上下限计算。
- [ ] 估值区间允许参考未来增长率进行人工调整。
- [ ] 标记数据截止日期、预测期和缺失值，不用旧数据伪装实时数据。

验收标准：

- [ ] 标的详情页能在一个页面查看行情、财务、估值和新闻。
- [ ] 外部服务不可用时保留最后一次成功数据并明确提示时间。

## 阶段 5：交易计划与 AI 指引

目标：将纪律、研究数据和当前持仓组合成可解释的交易计划。

### 交易计划

- [ ] 支持为自选股和持仓股新建、编辑、版本化交易计划。
- [ ] 记录计划周期、核心逻辑、触发条件、失效条件、目标和止损。
- [ ] 计划状态支持草稿、待执行、执行中、完成、放弃和已失效。
- [ ] 计划与实际交易关联，便于检查是否按计划执行。

### AI 输入包

- [ ] 汇总适用交易纪律和历史复盘经验。
- [ ] 汇总近 3 年财务、估值、盈利预测和行业数据。
- [ ] 汇总重大新闻、情绪、资金流和技术面数据。
- [ ] 汇总当前持仓、账户现金、集中度和资产配置约束。
- [ ] 对发送到外部模型的数据进行脱敏并保存输入快照。

### AI 结构化输出

- [ ] 分别估算中短期波段胜率和长期价值投资胜率。
- [ ] 显示先验概率、更新条件、后验概率及不确定性，不把主观判断伪装成精确事实。
- [ ] 分别估算波段赔率和长期价值赔率。
- [ ] 使用分数凯莉计算建议仓位，并设置家庭级最大仓位和单标的上限。
- [ ] 生成金字塔建仓和减仓表：次数、价格、金额、累计仓位和触发条件。
- [ ] 输出继续持有、等待、减仓、清仓或放弃的建议及证伪条件。
- [ ] 保存模型、提示词版本、输入数据版本和生成时间。
- [ ] 支持人工修改并保留 AI 原始结果。

验收标准：

- [ ] 同一份输入可以复现对应的 AI 结果记录。
- [ ] 建议中的每项关键数字都能追溯到输入或明确标注为模型判断。
- [ ] AI 服务失败不会影响交易、持仓和手工计划功能。

## 阶段 6：资产配置分析

目标：把投资组合与家庭整体资产配置放在同一视角下。

- [ ] 复用家庭资产快照，避免重复录入同一笔投资资产。
- [ ] 建立家庭资产配置模板：类别、目标比例、允许上下限。
- [ ] 展示实际配置与目标配置饼图及差异。
- [ ] 按成员、账户、币种、地区和资产类别下钻。
- [ ] 结合可用现金、集中度和计划交易给出调整缺口。
- [ ] 调整建议区分“规则计算”和“AI 解读”。
- [ ] 保存每期配置结果，支持历史趋势对比。

验收标准：

- [ ] 任一资产快照都能与指定模板进行可复核的比例对比。
- [ ] 投资账户资产不会在家庭资产总额中重复计算。

## 阶段 7：外部接口与自动同步

目标：在不影响核心账务可用性的前提下，逐步自动化数据采集。

- [ ] 定义行情、财务、新闻、汇率和 AI 的统一适配接口。
- [ ] Futu 接入先完成 NAS/局域网部署可行性验证。
- [ ] 使用管理命令和 NAS 定时任务完成首版同步，不提前引入复杂任务队列。
- [ ] 同步任务支持超时、重试、幂等、限流和失败日志。
- [ ] 保留人工录入、CSV 导入和手动刷新作为降级路径。
- [ ] 增加数据源状态页：最近成功时间、失败原因和待处理数量。
- [ ] API 密钥只存环境变量，不写入数据库、日志或前端。

验收标准：

- [ ] 任一外部服务中断时，核心账务和历史查询仍可使用。
- [ ] 同步结果可追踪来源，并能安全重跑。

## 阶段 8：策略回测（后续）

目标：在交易和行情历史数据稳定后，再验证规则化策略。

- [ ] 定义首批回测策略，例如入通交易或金字塔交易。
- [ ] 明确标的池、时间区间、基准指数和调仓规则。
- [ ] 处理复权、停牌、涨跌停、手续费、税费和滑点。
- [ ] 防止未来函数和幸存者偏差。
- [ ] 输出累计收益、最大回撤、胜率、盈亏比和交易明细。
- [ ] 保存策略版本、参数和数据版本，确保结果可复现。
- [ ] 回测结果可转为模拟计划，但不得直接触发真实交易。

验收标准：

- [ ] 同一策略、参数和数据版本重复运行结果一致。
- [ ] 回测报告明确列出假设、限制和数据缺口。

## 四、推荐迭代顺序

1. 阶段 0：确认核算规则。
2. 阶段 1：交易流水与持仓核算引擎。
3. 阶段 2：持仓、交易和仪表盘。
4. 阶段 3：交易日志、纪律与复盘。
5. 阶段 4：自选股、行情、新闻、财务和估值。
6. 阶段 5：AI 交易计划和金字塔建议。
7. 阶段 6：家庭资产配置分析。
8. 阶段 7：完善自动同步和运维。
9. 阶段 8：策略回测。
